[02] JVM BYTECODE
What javac produces (.class file)
idle
#0 iload_1 push local var a onto operand stack
#1 iload_2 push local var b onto operand stack
#2 iadd pop 2, add ints, push result ← stack-based
#3 istore_3 pop into local var c
Tầng này: Bytecode là instruction của máy ảo JVM, không phải CPU thật. Stack-based: không có register, mọi thứ qua operand stack. File .class chạy được trên mọi OS/CPU vì JVM trừu tượng hoá phần cứng.
▼ JIT (HotSpot)
[03] JIT COMPILER · HotSpot C2
JVM dịch bytecode → native khi "hot"
idle
iload_1 + iload_2 + iadd → ADD EAX, EBX stack ops gộp & map vào register thật
istore_3 → MOV [c], EAX ghi register về memory slot của c
Tầng này: Ban đầu JVM interpret bytecode (chậm). Khi method chạy nhiều lần (≥ ~10k lần) → JIT (C1 rồi C2) dịch sang native x86/ARM + optimize (inlining, dead code elim, register allocation). Đây là lý do Java "nóng máy" mới nhanh.

[04] CPU INSTRUCTIONS · x86
What the CPU executes
idle
0x1000 MOV EAX, [a] load a (=3) into register EAX
0x1004 MOV EBX, [b] load b (=5) into register EBX
0x1008 ADD EAX, EBX EAX ← EAX + EBX ← THE ACTUAL ADDITION
0x100C MOV [c], EAX store result (=8) into c
Tầng này: Mỗi dòng code phân rã thành nhiều instruction đơn giản. ADD là instruction quan trọng — nó kích hoạt ALU (Arithmetic Logic Unit) bên trong CPU.

[05] LOGIC GATES · HALF ADDER
How ADD physically works
idle
A (binary) 011
B (binary) 101
Processing bit —
A[i] XOR B[i] → SUM —
A[i] AND B[i] → CARRY —
Result (binary) 000
Tầng này: ADD = cộng từng cặp bit. Mỗi bit dùng một half adder = 1 cổng XOR (cho SUM) + 1 cổng AND (cho CARRY). Cộng 3 + 5 trong binary: 011 + 101 = 1000. Animation đi từng bit từ phải sang trái.
